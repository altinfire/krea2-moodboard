## Design defaults (fit the shipped design; adjust with reason — write `adr-002` if you
diverge or settle the open questions differently)

- The tray IS the mood editor. A mood is `{id, name, desc, tags[], createdAt}` in
  localStorage `krea-mb-user-moods` (new key; do not rename `krea-mb-favs`/`krea-mb-tray`
  — live visitors already have them populated).
- **Custom tags** = a small text input in the tray that adds a typed chip to the working
  set; it then saves with the mood and copies out with Copy like any other chip.
- **Save as mood** = a button in the tray; name entry via an inline input — NEVER
  `window.prompt()` (modal dialogs hang the Chrome automation extension, and it's ugly).
- Rail: new "your moods" `.ix-group` beneath "collection"; one row per mood (name + tag
  count); click loads that mood's tags into the tray (replace, then re-sync pill
  highlights); a delete affordance (rename too if cheap). Hide the group while empty.
- Open questions you may settle: should clicking a mood also filter the board grid by
  fuzzy keyword match — lean NO for v1 (tray-load only); merge-vs-replace on load — lean
  replace.

## Gotchas / hard-won facts

- The §4 template is a PowerShell **single-quoted here-string**: `$` is safe, but no
  template line may begin with `'@`, and don't write `__TOKEN__`-shaped strings — only
  `__COUNT__` / `__DATA__` / `__BUILDDATE__` are meant to substitute, and `.Replace()`
  hits every occurrence including comments.
- The build inlines data with `-replace '</','<\/'` before `.Replace('__DATA__', ...)` —
  keep that escaping; and never write a literal `</script>` inside the template's JS.
- `-HtmlOnly` suffices for template-only work. If you touch §2 (catalog shape), rebuild
  with `-SkipImages` (re-crawls ~1 min); if the crawl finds NEW boards, run
  `./classify-new-boards.ps1`, then `./classify-mood-registers.ps1`, then rebuild again.
- Chrome automation on this 3.6 MB page: wait ≥5s after navigation; coordinate clicks
  silently miss before the JS binds — verify state actually changed after every click.
  The extension could NOT resize the browser window last session (innerWidth stayed
  pinned) — test the narrow layout by temporarily rewriting the 980px media query via
  CSSOM (`document.styleSheets[0]` → `r.media.mediaText = '(max-width: 1300px)'`, then
  reload to restore), or use headless Chrome. Headless is also how screenshot.png is
  made: `chrome --headless=new --window-size=1440,900 --virtual-time-budget=15000
  --screenshot=<path> http://localhost:8123/index.html`.
- NEVER trigger `alert` / `confirm` / `prompt` while driving Chrome — a modal dialog
  hangs the automation extension for the rest of the session.
- localStorage on the live origin already carries visitors' favorites and tray — parse
  defensively (the `load()` helper try/catches) and keep existing keys stable.
- Usage discipline: `node ~/.claude/check-usage.js`; at 100% the session dies without
  regard to in-flight state. This feature is cheap; the spike's model download is
  Chrome-side, not API tokens.
- git warns LF→CRLF on the md files — harmless.
